基础

interface（接口）

接口是对 JavaScript 本身的随意性进行约束，在声明一个对象、函数或者类时，先定义接口，确保其数据结构的一致性。

在多人协作时，定义接口尤为重要。

TypeScript 编译器依赖接口用于类型检查，最终编译为 JavaScript 后，接口将会被移除。

interface Clothes {
    func(): void; //函数
    color?: string; //可选属性
    readonly size: string; //只读属性
    price: number;
    [key: string]: any; //任意属性
}
let arrO: ReadonlyArray<number> = [1, 2, 3, 4, 5]; //只读数组
function getClothesInfo(clothes: Clothes): void {
    console.log("interface", clothes.price);
}
let myClothes: Clothes = {
    func: function () {
        console.log("123");
    },
    //   color: "black",
    size: "XL",
    price: 98,
};
getClothesInfo(myClothes);

关于interface接口

1.定义接口要 首字母大写
2.定义的变量比接口少了一些属性或者多一些属性都是不允许的
3.只需要关注值的 外形，并不像其他语言一样，定义接口是为了实现

enum(枚举)

本节介绍枚举类型的定义及其使用，需要定义一组相同主题的常量数据时，应该立即想到枚举类型。在学习过程中，需要注意枚举类型的正向映射和反向映射，可以通过编译后的 JavaScript 源码进行分析，为什么可以进行反向映射。

//ts
enum Controls {
  Text,
  Number,
  Date,
}

//js
var Controls;
(function (Controls) {
    Controls[Controls["Text"] = 0] = "Text";
    Controls[Controls["Number"] = 1] = "Number";
    Controls[Controls["Date"] = 2] = "Date";
})(Controls || (Controls = {}));

关于枚举

1.声明一个枚举类型，如果没有赋值，它们的值默认为数字类型且从 0 开始累加

enum Months {
  Jan,
  Feb,
  Mar,
  Apr
}
Months.Jan === 0 // true
Months.Feb === 1 // true
Months.Mar === 2 // true
Months.Apr === 3 // true

class(类)

类描述了所创建的对象共同的属性和方法。通过 class 关键字声明一个类，主要包含以下模块：

  • 属性
  • 构造函数
  • 方法
    class BaseModel {
      public code: number;
      public message: string;
      public data?: object;
      public private() :void{
        console.log('private')
      }
      constructor(code: number, message: string) {
        this.code = code;
        this.message = message;
        this.private();
      }
    }
    class SuccessModel extends BaseModel {
      constructor(code: number, message: string, data: object) {
        super(code, message);
        this.data = data;
      }
    }
    class ErrorModel extends BaseModel {
      constructor(code: number, message: string, data?: object) {
        super(code, message);
        if (data) this.data = data;
      }
    }
    const baseModel = new BaseModel(100, "请求成功");
    const successModel = new SuccessModel(100, "请求成功", [1, "2", 3]);
    //{code: 100, data: (3) [1, 2, 3], message: "请求成功"}
    const errorModel = new ErrorModel(101, "请求失败");
    //{code: 101, message: "请求失败"}
    console.log("class", baseModel, successModel, errorModel);

类型保护

通过 typeof、instanceof、in 和 字面量类型 将代码分割成范围更小的代码块，在这一小块中，变量的类型是确定的

  function judgeType(param: number | string | boolean): void {
    if (typeof param === "number") {
        console.log('类型保护','number')
    }
    if (typeof param === "string") {
        console.log('类型保护','string')
    }
    if (typeof param === "boolean") {
        console.log('类型保护','boolean')
    }
  }
  judgeType('ok')

类型推断

类型推断主要用于那些没有明确指出类型的地方帮助确定和提供类型，这是 TypeScript 的一种能力。

类型推断是有方向的，要注意区分从左向右和从右向左两种推断的不同应用。

const str = 'aaa' //typescript 会推断为 const str:string = 'aaa'

类型断言

TypeScript 允许你覆盖它的推断，毕竟作为开发者你比编译器更了解你写的代码。

类型断言主要用于当 TypeScript 推断出来类型并不满足你的需求，你需要手动指定一个类型。

//error
const user = {}  //编译器推断 const user: {}，这是一个没有属性的对象，所以你不能对其添加属性。
userInfo.name="geyujie"  //Property 'name' does not exist on type '{}'

//success
interface User {
    [key:string]:any
}
const userInfo = {} as User
// const userInfo = <User>{}
userInfo.name="geyujie"  // ok
